Detailed Description

Memory pool implementation.

Provides a flexible yet predictible way to allocate memory. Fixed size memory blocks are allocated. Each block is always present in either of two linked lists, taken or free. Taking a block moves the block from the free list to the taken list in a predictible way, with no memory allocation done. Freeing a block moves it back to the free list.
