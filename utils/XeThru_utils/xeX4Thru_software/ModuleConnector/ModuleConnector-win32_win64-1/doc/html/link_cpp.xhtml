Project brief
Compiling and linking ModuleConnector with C++

Requirements for using the C++ interface to ModuleConnector:

  • Microsoft Windows 10 / Raspbian 8 (jessie) / Ubuntu 16.04 (glibcxx >=3.4.20) / OSX
  • XeThru Hardware Module, supported platforms are: X2M200, X4M200 and X4M300
  • XeThru ModuleConnector SW package
  • GCC 5/mingw compatible C++ compiler and linker with posix thread model. On Windows i686 the dwarf2 (dw2) exception handling method is used, on Windows x86_64, Structured Exception Handling (seh).

Other versions than the listed may also work but have not been tested.

Compiling and linking

ModuleConnector is distributed as a shared library. The libraries are located in the lib subfolder.

Compiling

To compile your code with ModuleConnector you have to include the path to the header files. 

g++ -I${PATH_TO_INCLUDE_DIR} -c ping.cpp -o ping.o

where ${PATH_TO_INCLUDE_DIR} evaluates to a relative or an absolute path to the ModuleConnector include directory.

Linking

When linking your application with ModuleConnector you have to link with the shared library. 

 g++ ping.o -o ping -L${PATH_TO_LIB_DIR} -lModuleConnector

Where ${PATH_TO_LIB_DIR} evaluates to a relative or absolute path to the ModuleConnector lib directory. On windows both 32bit and 64bit versions are included, with names lib32 and lib64.

Example toolchain setup on Windows

  1. Install and update http://www.msys2.org/ as per the instructions on that page.
  2. Install mingw64 compiler and make: 
    $ pacman -S mingw64/mingw-w64-x86_64-gcc mingw64/mingw-w64-x86_64-make
  3. Add the compiler to the path: 
    $ export PATH="/mingw64/bin:$PATH"
  4. Compile examples 
    $ cd path/to/unpacked/ModuleConnector-win32_win64/examples
$ make
...
  5. Run: 
    $ ./ping_x4m300.exe COM11
...
